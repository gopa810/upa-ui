1         Universal Platform Application

 

1.1        Architecture

UPA has two main components:

         Application backend

         User Interface

 

1.2        Communication UI with APP

 

UI makes these requests to APP:

         Get HTML page

         Web service call

 

1.3        Web-service communication with APP

 

Format of request:

JSON with format of array. Sequence of commands and arguments in one array.

[cmd1, arg1, arg2, argN]

 

Format of response:

JSON with format of array of arrays. Topmost array is collection of commands, individual records are commands with arguments.

Example:

[ [ cmd1, arg1, arg2, arg3 ], [ cmd2, arg1 ], . [cmdN] ]

 

1.3.1        Command clrscr

 

Clears main element in UI, so nothing is presented in the client area.

[ 'clrscr' ]

1.3.2        Command setHTML

 

Sets given UI items into client area.

[ 'setHTML', data ]

Data is dictionary with these keys:

target-id

 

data

Value is array of UI elements that are converted into HTML DOM entities and set in innerHTML property of target-id element. If not present, then 'html' key is used. Each UI element is dictionary with structure defined in next paragraph.

html

HTML text to be set in innerHTML property of target-id element. If not present, then 'data' key is used.

 

1.4        UI element in data:

There are 3 possible formats for UI element data. They are distinguished by the presence or non-presence of respective keys. If the "tag" key is present, it is description of HTML element in JSON format. If the key "template" is present, then it contains name of HTML template, that is obtained for rendering HTML content.

1.4.1        Explicit Data

Dictionary with following keys:

tag

Mandatory. Tag name of element created from this record. "div", "object", "canvas", "img" are examples.

id

ID of node

style

Style copied into style property of element.

_items

Value is array of subitems. Can be empty array. Subitems have the same structure as this main UI element.

..

Actually all keys that does not start with underscore are copied into new element

 

1.4.2         Explicit HTML text

Dictionary with template:

template

Name of template, that is retrieved also from server

<param_N>

Value of given parameter, that is substituted in template.

 

 

Example of template:

{% script: custom1.js %}

<div id="{{ id }}" style="background:{{ background }};color: {{ color }}">

{{ text }}

</div>

 

Dictionary that will reference this template will look like for example:

There are some predefined templates in UI itself, if given template name is not among them, then UI will attempt to get HTML template from APP.
